1. Introduction

1.1 Purpose
The main purpose of this SRS document is to provide a detailed overview of our software NSUCLS, its parameters, and goals. This document describes the project's target audience and its user interface, hardware and software requirements. Moreover, this document describes the usage of our software and required functionality lucidly so that the clients and involved developers can easily comprehend the project and what to do if anything needs to be changed.
1.2 Document Conventions
Heading: Font Style: Bold, Font Size: 18
Sub Heading: Font Style: Bold, Font Size: 14
Explanations: Font Style: Normal, Font Size: 12
1.3 Intended Audience and Reading Suggestions
This document is intended for the clients, developer team, project manager, users, and documentation writers. From this SRS, readers can understand finer details of the project thoroughly. For instance, the clients will have clear instructions on how to handle the project (website/app). Most importantly, the developer team will be able to know every single detail of the software so that they can execute it accurately. Moreover, from this SRS, the project manager can estimate the required time and work load to finish this project, and thus plan everything accordingly. Lastly, the users will be provided with precise instructions on how to use the software’s applications in both app and website versions. This document explains who this software is for, how and where it will be used, and what it can accomplish. It is suggested that the SRS be read in order using the table of content provided above.
1.4 Product Scope
The purpose of NSUCLS is to create an online platform for the members of NSU to complain if they face any issue. NSUCLS can be used to lodge a complaint against NSU staff, faculty or students. The complaint lodged will then be instantly reviewed by a faculty or the system admin. The main aim of NSUCLS is to ensure a faster and easier approach for the members of NSU to complain and fix that issue. NSUCLS is also very user friendly for the helpers as it provides special features like lodging complaints by Bangla speech recognition.
1.5 References
ExpressJS ReactJS MySQL

2. Overall Description

2.1 Product Perspective
NSUCLS is a new self-contained product. It mainly originated from the need of lodging complaints in a systematic way. This software makes the whole system, from lodging a complaint to solving that case, very efficient. Also, it prevents/stops false accusations as proper evidence must be presented with a reviewer to review them.
2.2 Product Functions
The software's function are user friendly so that it is easy for the users to operate. The major functions of the software are provided below.
  • All users (Students, TA, RA, Lab Instructor, Faculty, Helper, Admin) can lodge complaints.
  • ONLY Faculty or Admin can review complaints.
  • Complain lodger can edit and update his/her complaint.
  • The system admin can instantly create accounts, disable accounts, lodge complaint on behalf of any user.
2.3 User Classes and Characteristics
All users can access the software NSUCLS from two different platforms: Web and Android App.
There are three different types of actors, they are system Admin, reviewers and users. System admin can create an account in NSUCLS without the verification process by email. System admin can also disable an account and can lodge complaints on behalf of others. The reviewers are only faculty and system admin. They can both review and lodge complaints, whereas, other users can only lodge complaints but cannot review.
2.4 Operating Environment
The operating environment is given below:
  • Platform: Web and Android
  • Web Server: Any operation system with NodeJS RTE 16 or newer installed.
  • Android App: Any system with a modern web browser or for android with android version 7.0+
  • Database Configuration: MySQL
2.5 Design and Implementation Constraints
Only android is supported for now and users can also lodge complain using speech recognition. Users with android version less than 7 cannot use the app. The backend is developed using NodeJS and the frontend is developed using ReactJS. The android app is developed using Java.
2.6 User Documentation
The product will provide users with a user manual which will contain a section showing how to lodge a complain. It will also contain a section for the faculty members to show them the details of reviewing a complain.
2.7 Assumptions and Dependencies
Users must have a working internet. Web users must have a working browser to use the product. Phone users must have android version 7+ installed.

3. External Interface Requirements

3.1 User Interfaces
The user will be taken to the home page first. Users can sign up by their NSU mail and NSU ID. Only helpers can use their default mail for sign-up. Then users can login either by their mail or use google-sign in. After they are logged in, they can click "lodge a complaint" and will be redirected to that page. However, if they are not logged in, they will be redirected to the login page. Then, the users are required to fill up a form giving the necessary information such as attachments for evidence, selecting a proper reviewer and other users against whom the complaint is lodged. Users can lodge the complaint by speech (in Bangla) rather than text in the android application of the software by using the Bangla Speech recognition AI. After the complaint is finally lodged, an email will be sent to notify the lodger, reviewer and the system admin. Lastly, the lodger can change the reviewer and edit the complaint description.
3.2 Hardware Interfaces
The mobile app version of the software NSUCLS needs a microphone to input speech and convert it to text for lodging a complaint.
3.3 Software Interfaces
NSUCLS depends on the following software components:
  • ExpressJS - Express.js or simply Express, is a backend web application framework for Node.js designed for building web and mobile applications and APIs. The backend of NSUCLS is built using this framework.
  • ReactJS - React is a free and open-source front-end JavaScript library for building user interfaces based on UI components. The frontend or UI of NSUCLS is built using this library.
  • MySQL - MySQL is an open-source relational database management system. The data of NSUCLS will be stored in this DBMS.
  • Sequelizer - Sequelize is a promise-based Node.js ORM(Object Relational Mapper) for MySQL. This is used to avoid writing raw SQL queries.
  • Android framework- The android framework is the set of API's that allows to quickly and easily write apps for android phones. The mobile app version of the software will be built using this.
3.4 Communications Interfaces
The web application will send data to the backend using rest API. The android application will also communicate through its rest API. For android, an AI speech recognition will be used for Bangla speech to text conversion . All these communications will happen via HTTP. Authentication for google sign up is done by google and for normal sign up is done using their own authentication method.

5. Other Nonfunctional Requirements

5.1 Performance Requirements
Mobile App: Needs to be lightweight and responsive.
Database: The queries for DBMS data should be optimized as there would be a huge number of users.
Speech to text recognition: The AI used for this feature must be up to date. Inability of correct transformation to text might lead to serious confusions.
5.2 Safety Requirements
All data must be stored and never deleted from the database. For instance, if a lodger edit the complaint description or change reviewer, the previous data should be kept along with the new one in the database.
5.3 Security Requirements
The complaints will be confidential to other users who are not involved with that lodged complaint. The application should not be accessed by any unregistered account. Each registered account must be authorized with an email-verification process while logging in for the first time. Anyone outside NSU members cannot create an account. Any user other than system admin cannot access the admin page by any means. The system shall not disclose any personal information about the users.
5.4 Software Quality Attributes
User-friendliness: This software is designed particularly keeping the helpers of NSU in mind. Features like lodging a complaint via speech in bangla would make it easy for them to make use of this application. Since they don’t have nsu-mail, google sign-in process is made available.
RELIABILITY: This software is made secure so that personal information of the users won’t be disclosed. Moreover, the lodged complaints are kept confidential to the general users. Thus users will feel safe to use this application.
Flexibility: The software has both web and mobile app versions, thus making it flexible to use. In addition, the mobile app version has some special features as discussed above.
Availability: It is an online system and available 24/7 to all users.
5.5 Business Rules
The product will be free to use for all the members of NSU. No one outside NSU members can use this product.
